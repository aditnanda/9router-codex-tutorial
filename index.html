Tutorial Setup 9Router Proxy untuk Codex GUI di macOS & Windows

Panduan ini menjelaskan cara setting 9Router dari awal sampai bisa dipakai sebagai endpoint untuk Codex GUI, Codex Desktop, VS Code Extension, atau Cursor di macOS dan Windows.

Endpoint: https://domainkamu.com/v1 Combo: openai Model: cx/gpt-5.5 Provider: codex OS: macOS + Windows

Penting: Jangan Upload Token Asli

Jangan pernah membagikan file backup 9Router mentah ke GitHub, Gist, forum, atau chat publik. File backup bisa berisi accessToken, refreshToken, idToken, API key, email akun, dan account ID.

Tutorial ini memakai placeholder domainkamu.com dan sk-xxxxxxxxxxxxxxxx. Ganti dengan domain dan API key milik kamu sendiri saat praktik.

4. Login Provider Codex di 9Router

Buka dashboard:

https://domainkamu.com/dashboard

Masuk ke menu:

Providers

Tambahkan provider baru:

Provider : codex
Auth     : OAuth

Login menggunakan akun ChatGPT/Codex kamu.

Target Status

Provider harus aktif dengan status: active 

provider : codex
authType : oauth
status   : active

Jika punya beberapa akun Codex, kamu bisa menambahkan beberapa provider connection. 9Router dapat memakai fallback atau round-robin antar akun tersebut.

5. Buat Combo openai

Masuk ke menu:

Combos

Klik:

Create Combo

Isi combo seperti ini:

Combo name : openai
Model      : cx/gpt-5.5

Hasilnya kurang lebih:

{
  "name": "openai",
  "models": [
    "cx/gpt-5.5"
  ]
}

Kenapa nama combo-nya openai?

Karena Codex nanti cukup diarahkan ke model bernama openai. Jika suatu saat ingin mengganti model utama atau menambah fallback, cukup ubah combo di 9Router, tanpa perlu mengubah config Codex di Mac atau Windows.

6. Buat API Key di 9Router

Masuk ke menu:

Endpoint

Pada bagian API Keys, klik:

Create Key

Isi nama key:

openai

Contoh format API key:

sk-xxxxxxxxxxxxxxxx

Jika ingin endpoint wajib API key, aktifkan:

Require API key: ON

Request ke 9Router nantinya harus memakai header:

Authorization: Bearer API_KEY_KAMU

7. Setting Strategy dan Token Saver

Provider Strategy

Untuk provider codex, gunakan:

Provider          : codex
Fallback strategy : round-robin
Sticky limit      : 1

Combo Strategy

Untuk combo openai, gunakan:

Combo             : openai
Fallback strategy : round-robin

Token Saver

Rekomendasi setting:

Compress tool output / RTK : ON
Compress LLM output        : OFF
  • RTK membantu menghemat token dari output tool seperti git diff, ls, tree, log, dan grep.
  • Caveman sebaiknya dimatikan agar jawaban code, error, dan warning tetap natural.

8. Test Model dari Terminal

macOS

Terminal
Set API key sementara

curl https://domainkamu.com/v1/models \

  "owned_by": "combo"

Dan model langsung:

  "id": "cx/gpt-5.5",
  "object": "model",
  "owned_by": "cx"
}

9. Test Responses API

Codex GUI membutuhkan endpoint yang kompatibel dengan Responses API. Karena itu kita perlu memastikan endpoint /v1/responses bisa menerima request.

macOS

Terminal 
curl -s https://domainkamu.com/v1/responses \
  -H "Authorization: Bearer $NINEROUTER_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "openai",
    "input": "jawab satu kata: ok"
  }' | python3 -m json.tool

Windows

PowerShell 
$body = @{
  model = "openai"
  input = "jawab satu kata: ok"
} | ConvertTo-Json

curl.exe -s https://domainkamu.com/v1/responses `
  -H "Authorization: Bearer $env:NINEROUTER_API_KEY" `
  -H "Content-Type: application/json" `
  -d $body | python -m json.tool

Jika berhasil, akan muncul JSON response dari model.

Jika muncul error seperti ini:

404 not found
unsupported endpoint
Cannot POST /v1/responses

Berarti endpoint /v1/responses belum kompatibel. Jika /v1/models berhasil tetapi Codex GUI stuck di reconnecting, biasanya masalahnya ada di /v1/responses atau streaming.

10. Config Codex GUI: macOS dan Windows

macOS

~/.codex/config.toml

Codex membaca konfigurasi dari:

~/.codex/config.toml

Buat config:

mkdir -p ~/.codex

cat > ~/.codex/config.toml <<'EOF'
model = "openai"
model_provider = "ninerouter"
model_reasoning_summary = "none"
model_supports_reasoning_summaries = false

[model_providers.ninerouter]
name = "9Router"
base_url = "https://domainkamu.com/v1"
env_key = "NINEROUTER_API_KEY"
wire_api = "responses"
supports_websockets = false
stream_idle_timeout_ms = 600000
stream_max_retries = 10
EOF

chmod 600 ~/.codex/config.toml

Windows

%USERPROFILE%\.codex\config.toml

Codex membaca konfigurasi dari:

%USERPROFILE%\.codex\config.toml

Buat config via PowerShell:

New-Item -ItemType Directory -Force "$env:USERPROFILE\.codex" | Out-Null

@'
model = "openai"
model_provider = "ninerouter"
model_reasoning_summary = "none"
model_supports_reasoning_summaries = false

[model_providers.ninerouter]
name = "9Router"
base_url = "https://domainkamu.com/v1"
env_key = "NINEROUTER_API_KEY"
wire_api = "responses"
supports_websockets = false
stream_idle_timeout_ms = 600000
stream_max_retries = 10
'@ | Set-Content -Encoding UTF8 "$env:USERPROFILE\.codex\config.toml"

Penjelasan Config

model = "openai"

Menggunakan combo openai dari 9Router.

model_provider = "ninerouter"

Menggunakan custom provider bernama ninerouter.

base_url = "https://domainkamu.com/v1"

Endpoint 9Router.

env_key = "NINEROUTER_API_KEY"

Nama environment variable untuk API key.

wire_api = "responses"

Memaksa Codex memakai Responses API.

supports_websockets = false

Agar Codex tidak memaksa websocket.

stream_idle_timeout_ms = 600000
stream_max_retries = 10

Membantu mengurangi masalah reconnect saat streaming.

11. Set API Key: macOS dan Windows

macOS

launchctl + zsh

Aplikasi GUI di macOS sering tidak membaca environment variable dari .zshrc. Karena itu set API key memakai launchctl. 

launchctl setenv NINEROUTER_API_KEY "ISI_API_KEY_9ROUTER_KAMU"

Supaya Terminal juga membaca API key:

grep -q 'NINEROUTER_API_KEY' ~/.zshrc || echo 'export NINEROUTER_API_KEY="ISI_API_KEY_9ROUTER_KAMU"' >> ~/.zshrc
source ~/.zshrc

Cek:

echo $NINEROUTER_API_KEY

Windows

User Environment Variable

Untuk Windows GUI, set environment variable level user agar terbaca oleh VS Code, Cursor, atau aplikasi Codex setelah direstart. 

[Environment]::SetEnvironmentVariable(
  "NINEROUTER_API_KEY",
  "ISI_API_KEY_9ROUTER_KAMU",
  "User"
)

Set juga untuk sesi PowerShell saat ini:

$env:NINEROUTER_API_KEY="ISI_API_KEY_9ROUTER_KAMU"

Cek:

echo $env:NINEROUTER_API_KEY

Catatan Windows

Setelah memakai [Environment]::SetEnvironmentVariable, tutup semua jendela VS Code/Cursor/Codex dan buka ulang agar env baru terbaca.

13. Troubleshooting

A. /v1/models berhasil tapi Codex GUI reconnecting

Gejala:

Reconnecting... 2/5
Reconnecting... 3/5
Thinking

Penyebab umum:

  • /v1/responses belum kompatibel.
  • Streaming response putus.
  • Websocket dipakai padahal proxy tidak support.
  • API key tidak terbaca oleh aplikasi GUI.

Pastikan config memakai:

wire_api = "responses"
supports_websockets = false
stream_idle_timeout_ms = 600000
stream_max_retries = 10

macOS

Cek ulang 
cat ~/.codex/config.toml
echo $NINEROUTER_API_KEY

Windows

Cek ulang 
Get-Content "$env:USERPROFILE\.codex\config.toml"
echo $env:NINEROUTER_API_KEY

B. Hapus / ganti API key

macOS

launchctl 
launchctl unsetenv NINEROUTER_API_KEY
launchctl setenv NINEROUTER_API_KEY "API_KEY_BARU"

Windows

PowerShell 
[Environment]::SetEnvironmentVariable("NINEROUTER_API_KEY", $null, "User")
[Environment]::SetEnvironmentVariable("NINEROUTER_API_KEY", "API_KEY_BARU", "User")
$env:NINEROUTER_API_KEY="API_KEY_BARU"

C. Jika memakai endpoint lokal

Jika Codex dan 9Router berjalan di komputer yang sama, bisa pakai endpoint lokal.

macOS

config.toml 
base_url = "http://127.0.0.1:20128/v1"
curl http://127.0.0.1:20128/v1/models \
  -H "Authorization: Bearer $NINEROUTER_API_KEY" | python3 -m json.tool

Windows

config.toml 
base_url = "http://127.0.0.1:20128/v1"
curl.exe http://127.0.0.1:20128/v1/models `
  -H "Authorization: Bearer $env:NINEROUTER_API_KEY" | python -m json.tool

15. Deploy ke GitHub Pages

Buat repository baru, misalnya:

9router-codex-tutorial

Buat file:

index.html

Paste semua isi HTML tutorial ini ke file tersebut, lalu commit.

Aktifkan GitHub Pages

Repository → Settings → Pages
Source     → Deploy from a branch
Branch     → main
Folder     → / root
Save

URL akan menjadi:

https://USERNAME.github.io/9router-codex-tutorial/

Custom Domain

Jika ingin memakai subdomain, contoh:

docs.domainkamu.com

DNS:

Type  : CNAME
Name  : docs
Value : USERNAME.github.io

Lalu di GitHub Pages, isi custom domain:

docs.domainkamu.com

Catatan Keamanan

Jangan upload file backup 9Router mentah karena bisa berisi:

accessToken
refreshToken
idToken
API key
email akun
account id

Jika file backup atau API key sudah pernah terekspos publik:

  1. Hapus Gist/file publik.
  2. Buat API key baru di 9Router.
  3. Hapus API key lama.
  4. Re-login provider Codex jika perlu.
  5. Update env macOS atau Windows.

macOS

rotate env 
launchctl setenv NINEROUTER_API_KEY "API_KEY_BARU"

Windows

rotate env 
[Environment]::SetEnvironmentVariable(
  "NINEROUTER_API_KEY",
  "API_KEY_BARU",
  "User"
)
